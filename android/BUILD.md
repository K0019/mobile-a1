# Android Build Guide

## Quick Start (from repo root)

```powershell
# First time only — initialize submodules:
scripts\setup_submodules.bat

# Build, install, and run:
powershell -ExecutionPolicy Bypass -File scripts\build_android.ps1
```

The script auto-detects Java, Python, and the Android SDK, then handles asset
manifest generation, ASTC texture recompression, Gradle build, and APK install.

---

## Prerequisites

| Tool | How to get it |
|------|---------------|
| **Android Studio** | Provides SDK, NDK, and bundled JBR (Java) |
| **Android SDK** (API 30+) | Via Android Studio SDK Manager |
| **Android NDK** r29+ | Via Android Studio SDK Manager |
| **Python 3** | [python.org](https://python.org) — must be on `PATH` |
| **ADB** | Included with Android SDK Platform-Tools |

> **Note:** `local.properties` (SDK path) is auto-generated by `build_android.ps1`
> if it doesn't exist. You can also create it manually:
> ```
> sdk.dir=C\:\\Users\\YourName\\AppData\\Local\\Android\\Sdk
> ```

## Build Options

```powershell
# Skip asset steps if only code changed:
scripts\build_android.ps1 -InstallOnly

# Skip ASTC recompression (textures unchanged):
scripts\build_android.ps1 -SkipAssetCompile

# Release build:
scripts\build_android.ps1 -BuildType Release
```

## How the Asset Pipeline Works

The Gradle build (`android/app/build.gradle`) automatically:

1. **Generates `asset_manifest.txt`** — maps lowercase paths to actual file paths
   for case-insensitive VFS lookup on Android.
2. **Recompresses textures to ASTC** — converts desktop BCn-format KTX2 textures
   to ASTC 4x4 for mobile GPUs (requires `Tools/Release/AssetCompiler.exe`).
3. **Packages assets from `Assets/`** directly — no manual copying needed.
   Desktop-only compiled assets (`CompiledAssets/` except `CompiledAssets/android/`)
   are excluded to keep the APK under the 2 GB ZIP limit.

## Debugging

```bash
# All engine logs
adb logcat -s ryEngine

# Vulkan validation errors
adb logcat | grep -i validation

# Touch input
adb logcat | grep -i TOUCH
```

## Known Issues

- **Case sensitivity**: Android's AAssetManager is case-sensitive. The VFS uses
  `asset_manifest.txt` for case-insensitive lookup. If you add new assets, the
  manifest is regenerated automatically at build time.
- **ASTC textures**: Desktop textures (BC7/BC5) don't work on mobile GPUs.
  The build pipeline recompresses them to ASTC. If `AssetCompiler.exe` hasn't
  been built yet, this step is skipped with a warning.
